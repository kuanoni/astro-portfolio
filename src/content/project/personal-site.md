## Summary

This site, the one you're reading this on, is made up of 2 parts:

- Portfolio: A place where I can display my programming skills and web development experience. I'm self-taught with no formal educational experience, and so it's important for me to showcase the work that I'm most proud of and that best reflects my abilities. 

- Personal: A place where I can show off other creative works of mine. I've always had a wide variety of interests and like the idea of having one central place for me to gather the fruits of those interests.

## The plan

The main function of this site was to display content (text, images, videos, etc.), so I knew that there would be a minimal (if any) amount of backend required, and a minimal amount of complex user interactivity. 
No authorization. No live fetching or cache validation.
Since I'm the singular developer and content creator, I figured that there wasn't really any point in having a CMS. It would simply introduce more complications and create a reliance on a service that I didn't really need.

I knew that the site would require essentially no backend or complicated UI elements, so a static site generator like Astro seemed like the perfect choice. Also, after working with React and Next for so long, I wanted to practice a bit more "vanilla" JavaScript.
