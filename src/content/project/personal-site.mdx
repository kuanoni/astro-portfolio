## Introduction

Welcome to my little corner of the internet. As a self-taught web developer with no formal education, I created this website to prove to the world that I can actually code (shocking, I know). In addition to showcasing my web development projects, I've also thrown in some creative works like songs and videos, just to keep things interesting.

To build this website, I used [Astro](https://astro.build/), a static site generator that turns my jumbled thoughts and markdown files into web pages that actually look decent.

## Goals and Objectives

### Development experience

The main function of this site is to serve statically generated content, so I knew that there would be minimal backend work needed (no authentication, no server-side rendering, no API, etc.)

I wanted the content generation to be as easy as possible on the developer side. There should be a simple and repeatable pipeline from writing to publishing.

### User experience

Overall, I knew that I wanted the user's experience of the site to be:

- **Fast** in both browser performance and loading times. That means minimal JavaScript, small file sizes, and performance optimized CSS animations (if any).
- **Accessible** to anyone who wants to use the site. It should follow the [WCAG](https://www.w3.org/WAI/standards-guidelines/wcag/) by using semantic HTML or ARIA attributes as needed and be keyboard navigable.   
- **Discoverable** with accessible content, OG/Twitter tags, and passing [Lighthouse](https://developer.chrome.com/docs/lighthouse/overview/) tests. 
- **Simple** in its visual design and UX. I took inspiration from sites like Wikipedia where each page can be easily understood and navigated at a glance. The user's experience should be cohesive, consistent, and predictable.
